## 具体任务

### 1. Agent 三大范式 (1.5h)

**ReAct** (Reasoning + Acting):
```
Thought → Action → Observation → Thought → ... → Final Answer
```
- 优点：灵活、通用
- 缺点：容易循环、没有长期规划
- 你的项目：simple_agent.py 实现的就是这个

**Plan-Execute**:
```
User Request → Planner (生成步骤列表) → Executor (逐步执行) → Replanner (按需调整)
```
- 优点：有全局规划、适合复杂任务
- 缺点：规划可能不准确、需要 replan
- 代表：Plan-and-Solve Prompting

**Reflexion**:
```
Act → Evaluate → Reflect (口述自省) → Retry with new knowledge
```
- 优点：自我改进、从失败中学习
- 缺点：多次 LLM 调用成本高
- 代表：Reflexion Agent

**面试答案框架**：每种范式 = 流程图 + 一句话定义 + 适用场景 + 局限

### 2. RAG 全链路 (1.5h)

```
文档 → 解析 → 分块 → Embedding → 向量库 → 检索 → Rerank → 拼接 Prompt → 生成
```

每步的优化方案（面试重点）：

| 环节 | 常见问题 | 优化方案 |
|------|----------|----------|
| 分块 | 大小不合适 | 语义分块 / 滑动窗口 / 父子文档 |
| Embedding | 领域适配差 | 微调 Embedding 模型 / 混合编码 |
| 检索 | 召回率低 | 混合检索(BM25+向量) / HyDE / 多轮改写 |
| Rerank | 相关文档排在后面 | Cross-encoder / LLM Rerank |
| 生成 | 幻觉 / 引用不准 | 引用溯源 / 答案校验 |

### 3. 框架对比 (1h)

| 学习曲线 | 中等 | 较高 | 中等 | 中等 |
