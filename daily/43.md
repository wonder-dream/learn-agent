## 具体任务

### 1. Python 高频 (2h)

**GIL (Global Interpreter Lock)**:
- 是什么：CPython 的互斥锁，同一时刻只有一个线程执行 Python 字节码
- 为什么有：简化内存管理（引用计数不需要锁）
- 影响：CPU 密集型多线程无效，IO 密集型不受影响
- 怎么绕过：multiprocessing / 用 C 扩展释放 GIL / 换 Jython/IronPython

**装饰器原理**:
```python
@decorator
def func(): pass
# 等价于: func = decorator(func)
# 装饰器就是一个接受函数、返回新函数的高阶函数
```

**生成器 vs 迭代器**:
- 迭代器：实现了 `__iter__` 和 `__next__` 的对象
- 生成器：用 `yield` 的函数，自动实现迭代器协议
- 优势：惰性求值，省内存（不需要一次性生成所有数据）

**asyncio 事件循环**:
- 协程 = 可暂停/恢复的函数
- 事件循环 = 调度协程的执行器（单线程，非抢占式）
- `await` = 主动让出控制权："我先不跑了，你跑"

**可变对象 vs 不可变对象**:
- 不可变：int, str, tuple — 不能就地修改
- 可变：list, dict, set — 可以就地修改
- 默认参数陷阱：`def f(x=[])` — 默认值只在定义时计算一次！

### 2. 操作系统 (2h)

**进程 vs 线程 vs 协程**:

| | 进程 | 线程 | 协程 |
|---|------|------|------|
| 调度 | OS | OS | 用户代码 |
| 切换成本 | 高 | 中 | 低 |
| 内存 | 独立 | 共享 | 共享 |
| 通信 | IPC | 共享内存 | 直接调用 |

**内存管理**:
- 虚拟内存：每个进程看到独立的地址空间
- 分页：物理内存按页(4KB)分配
- 缺页中断：访问不在物理内存的页 → OS 从磁盘换入

**IO 多路复用**:
- select / poll / epoll（Linux）/ kqueue（macOS）
- 一个线程同时监听多个 IO 事件
- FastAPI 底层就是用的这个（通过 uvloop）
