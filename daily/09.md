### 1. Function Calling 原理 (1h)

**Function Calling 不是让 LLM 真正执行代码**，而是——LLM 输出一个 JSON 说"我想调用这个函数，参数是这些"，你（开发者）执行这个函数，结果再喂回给 LLM。

```
User: "评审 src/utils.py"
→ LLM: 输出 {"name": "read_file", "arguments": {"path": "src/utils.py"}}
→ Developer: 执行 read_file("src/utils.py")，拿到 content
→ LLM: 拿到内容后给出代码评审结果
```

Agent 的核心循环就是：LLM 推理 → 输出 tool_call → 你执行 → 结果回传 → 继续推理。

### 3. 写一个完整的 Tool Calling 循环 (1h)

手写一个最简循环（不用框架，理解本质）：

```python
import json

def tool_loop(messages: list[dict]) -> str:
    """最简 Agent 循环：LLM 要调工具就执行，直到输出最终答案"""
    while True:
        response = client.chat.completions.create(
            model="deepseek-v4-pro",
            messages=messages,
            tools=TOOLS,
        )
        msg = response.choices[0].message

        if msg.tool_calls:
            # LLM 想调工具 → 执行工具 → 结果追加回 messages
            messages.append(msg.model_dump())  # LLM 的 tool_call 消息
            for tc in msg.tool_calls:
                result = execute_tool(tc.function.name, json.loads(tc.function.arguments))
                messages.append({
                    "role": "tool",
                    "tool_call_id": tc.id,
                    "content": json.dumps(result),
                })
            # 继续循环，让 LLM 看工具结果
        else:
            # LLM 输出最终文本答案
            return msg.content
```

这个循环是目前所有 Agent 框架（LangChain/AutoGen/LangGraph）的本质，不要被框架包装迷惑。
