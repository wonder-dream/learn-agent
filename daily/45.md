## 具体任务

### 1. 电梯演讲 (1h)

2 分钟讲清楚：**背景 → 动机 → 做法 → 亮点 → 结果**

模板：
> 我做了一个 AI 驱动的代码审查 Agent。市面上现有的代码审查工具要么只做静态规则检查（比如 pylint 只能查风格），要么完全依赖 LLM（幻觉多、成本高）。我的方案是**混合架构**——先用 Python AST 做静态筛查（11 条规则），筛出候选问题代码段，再让 LLM 做精判。整个流水线用 LangGraph 的状态图编排，支持 checkpoint 暂停恢复。在 10 个开源项目上测试，准确率 87%，单文件平均成本不到 3 分钱。

练习：对着镜子计时讲，控制在 2 分钟。

### 2. 10 个高频追问 (2h)

准备每个问题的 30-60 秒答案：

| # | 追问 | 回答要点 |
|---|------|----------|
| 1 | "为什么不用直接用 pylint/ruff？" | 只能做规则检查，不懂语义；我可以检测逻辑 bug |
| 2 | "为什么不全交给 LLM 做？" | 幻觉 + 成本 + 速度；静态层过滤后 LLM 只处理 20% 的代码 |
| 3 | "为什么用 LangGraph 而不是 LangChain？" | 审查流程是有状态的多步流水线，StateGraph 比线性 Chain 更合适 |
| 4 | "静态分析准确率多少？LLM 精判准确率多少？" | 有数据！静态 95% 准确率但只能查规则性问题，LLM 精判 80%（覆盖语义问题） |
| 5 | "你怎么处理 LLM 调用失败？" | 指数退避重试 + 超时降级（跳过该候选）+ 结构化错误日志 |
| 6 | "如果审查一个 5000 行的文件怎么办？" | 候选人提取机制确保 LLM 只看到可疑片段，不是全文件 |
| 7 | "多 Agent 怎么做？" | 当前是单 Agent 多步骤；架构预留了多 Agent 插槽，Style/Logic/Security 三个专家 Agent + 协调者 |
| 8 | "MCP 在你的项目里怎么用的？" | 把代码审查能力暴露为 MCP Server，其他 AI 编程工具可以通过标准协议调用 |
| 9 | "你怎么验证 LLM 的输出是准确的？" | 10 个真实文件的人工标注验证 + false positive/false negative 分析 |
| 10 | "如果让你重新做，会怎么做？" | 先做多 Agent 分工 + 接入 GitHub App 做成真实产品 |

### 3. "最大挑战"故事 (1h)

准备 3 个故事，每个 = 遇到的困难 + 尝试的方案 + 最终解决 + 学到的教训：

1. **AST 分析误报太多** → 加了 severity 分级 + 可配置的规则阈值
2. **LLM 输出格式不稳定** → 调 Prompt + 加 JSON schema 约束 + 解析失败兜底
3. **LangGraph 学习曲线陡** → 先用原生 Python 写了一遍理解本质，再用 LangGraph 迁移

### 4. 场景设计题准备 (0.5h)

面试常见："给你 XX 场景，设计一个 Agent 系统"

6. 提一下可观测性（trace + 成本）
