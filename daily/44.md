## 具体任务

### 1. MySQL (2h)

**索引（面试最高频）**:
- B+ 树 vs B 树 vs Hash 索引
- 聚簇索引 vs 非聚簇索引（回表查询）
- 联合索引的最左前缀原则
- 覆盖索引：查询列全在索引里，不用回表
- 索引失效场景：`LIKE '%x'`、类型转换、函数包裹

**事务 ACID + 隔离级别**:

| 隔离级别 | 脏读 | 不可重复读 | 幻读 |
|----------|------|------------|------|
| READ UNCOMMITTED | Y | Y | Y |
| READ COMMITTED | N | Y | Y |
| REPEATABLE READ | N | N | Y* |
| SERIALIZABLE | N | N | N |

*InnoDB 通过 Next-Key Lock 解决了幻读

**SQL 优化**:
- EXPLAIN 看执行计划
- 慢查询日志定位
- 分库分表：水平分表（按 ID hash）/ 垂直分表（按列拆分）

### 2. Redis (1h)

**5 种基础数据结构**:
- String：缓存、计数器、分布式锁（SETNX）
- Hash：对象存储
- List：消息队列（LPUSH + BRPOP）
- Set：去重、交并差
- Sorted Set：排行榜（ZADD + ZRANGE）

**缓存策略**:
- Cache Aside：先读缓存，miss 则读 DB 并回写
- 缓存穿透：布隆过滤器 / 空值缓存
- 缓存击穿：热点 key 过期 → 互斥锁 / 永不过期
- 缓存雪崩：大批 key 同时过期 → 随机过期时间

### 3. 网络 (1h)

**TCP 三次握手 + 四次挥手**:
- 背下来，能画图
- TIME_WAIT 为什么是 2MSL
- 为什么是三次不是两次

**HTTP**:
- 1.0 vs 1.1（长连接）vs 2.0（多路复用）vs 3.0（QUIC）
- HTTPS = HTTP + TLS：证书验证 + 密钥交换 + 对称加密
- 状态码：200/301/302/400/401/403/404/500/502/503

**DNS 解析过程**:
```
浏览器缓存 → hosts 文件 → 本地 DNS → 根域名服务器 → 顶级域名服务器 → 权威 DNS
```
