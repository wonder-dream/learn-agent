### 1. 条件路由进阶 (1h)

Day 12 的条件边只有两路（继续/停止）。真实审查流程需要更复杂的分支：

```python
from langgraph.graph import StateGraph, END
from typing import TypedDict, Literal

class ReviewState(TypedDict):
    source_code: str
    file_path: str
    ast_summary: dict       # AST 解析结果
    candidates: list        # 静态分析筛出候选问题
    review_comments: list   # LLM 确认的最终问题
    current_step: str       # 当前在哪个阶段

# 条件路由函数
def route_after_ast(state: ReviewState) -> Literal["static_check", "skip_static"]:
    """AST 解析完：如果有函数/类就进入静态检查，否则跳过"""
    if state["ast_summary"].get("functions") or state["ast_summary"].get("classes"):
        return "static_check"
    return "skip_static"

def route_after_static(state: ReviewState) -> Literal["llm_review", "end"]:
    """静态分析完：如果有候选问题就进入 LLM 精判，否则结束"""
    if state["candidates"]:
        return "llm_review"
    return "end"
```

**技巧**：条件路由就是状态机的状态转移函数——把每个转移逻辑独立出来，方便测试。

### 2. Checkpoint (1h)

```python
# LangGraph 内置 Checkpoint——长任务可暂停/恢复/回溯
from langgraph.checkpoint.memory import MemorySaver

checkpointer = MemorySaver()
graph = StateGraph(ReviewState)
# ... 加节点、边 ...
app = graph.compile(checkpointer=checkpointer)

# 运行到某一步，保存状态
config = {"configurable": {"thread_id": "review-001"}}
result = app.invoke(initial_state, config)

# 如果中断了，可以从 checkpoint 恢复继续
# （长审查任务可能耗时几十秒，中断恢复是关键体验）
```

**面试价值**：如果面试官问"你审查一个 5000 行的文件，中途 LLM 超时了怎么办？"
→ 答：LangGraph checkpoint 机制可以在每个节点后保存状态，中断后从最近的 checkpoint 恢复，不用从头再跑。
