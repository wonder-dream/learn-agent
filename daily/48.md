## 模拟面试流程

### Part 1: 项目介绍 (5 min)

讲代码审查 Agent：架构 → 技术选型 → 量化结果

### Part 2: 深挖追问 (30 min)

找一个人（或自己）按 Day 45 的 10 个追问 + 随机追问：

追问逻辑要层层递进：
```
"你为什么做这个项目？"
→ "为什么用 LangGraph 而不是直接写个 while 循环？"
→ "StateGraph 的节点怎么设计的？"
→ "如果某个节点失败了怎么办？"
→ "你的 checkpoint 具体怎么用的？"
→ "如果审查 1 万行代码，瓶颈在哪？"
```

面试官会一直追问到你回答不上来——然后看你怎么应对（诚实说不知道 vs 瞎编）。

### Part 3: 场景设计题 (20 min)

随机抽一个场景：

**场景 A**："设计一个代码审查的多 Agent 系统，一个 Agent 审查风格，一个审查逻辑，一个审查安全。如何分工和协作？"

**场景 B**："设计一个智能客服 Agent，能查知识库（RAG）、能调用工单系统（Tool Calling）、能升级到人工（Human-in-the-loop）。"

**场景 C**："如果你要把你的代码审查 Agent 从支持 Python 扩展到支持 10 种语言，架构怎么改？"

回答结构：
1. 画架构图（文字描述也行）
2. 确定 Agent 的数量和角色
3. 通信方式（直接调用、消息队列、共享 State）
4. 关键难点 + 解决方案
5. 可观测性 + 容错

- 场景设计哪个环节没说清楚？
