### 1. 理解 LangGraph 的设计哲学 (1h)

昨天的 Agent 循环是用 `while True` 暴力循环。问题：
- 状态散落在 messages 列表里，难以追踪
- 循环条件（什么时候停）硬编码在代码里
- 条件路由（调哪个工具）用 if-else 堆叠

LangGraph 的解决思路：**Agent 的执行流程是一个有向图**。

```
[START] → [call_llm] → {有 tool_call?}
                           ├── Yes → [execute_tool] → [call_llm]
                           └── No  → [END]
```

三要素：
- **State**：贯穿所有节点的共享状态（TypedDict 或 pydantic）
- **Node**：图中的每个处理单元（Python 函数）
- **Edge**：节点之间的连线（普通边 / 条件边）

### 2. 跑通第一个 LangGraph (1.5h)

```bash
pip install langgraph langchain-core
```

```python
# 最简 LangGraph 示例
from typing import TypedDict, Annotated
import operator
from langgraph.graph import StateGraph, END

# 1. 定义 State——图中流通的数据
class AgentState(TypedDict):
    messages: Annotated[list, operator.add]  # Annotated + operator.add = 追加而非覆盖
    tool_calls: list
    final_result: str

# 2. 定义 Node——处理函数
def call_llm(state: AgentState) -> AgentState:
    """调用 LLM，追加 assistant message 到 messages"""
    # 实际代码：调 LLM → 拿到 response
    new_message = {"role": "assistant", "content": "I received: " + str(state["messages"])}
    return {"messages": [new_message]}  # 追加，不覆盖

def execute_tools(state: AgentState) -> AgentState:
    """执行工具调用"""
    return {"messages": [{"role": "tool", "content": "tool result"}]}

# 3. 建图
graph = StateGraph(AgentState)

# 加节点
graph.add_node("llm", call_llm)
graph.add_node("tools", execute_tools)

# 加边
graph.set_entry_point("llm")

# 条件边：根据 LLM 输出决定去哪
def should_continue(state: AgentState) -> str:
    last_msg = state["messages"][-1]
    if "TOOL_CALL" in last_msg["content"]:
        return "tools"
    return END

graph.add_conditional_edges("llm", should_continue, {
    "tools": "tools",
    END: END,
})
graph.add_edge("tools", "llm")  # 工具执行完 → 回到 LLM

# 编译 + 运行
app = graph.compile()
result = app.invoke({"messages": [{"role": "user", "content": "Review this code"}], "tool_calls": [], "final_result": ""})
print(result)
```

**关键理解**：
- `Annotated[list, operator.add]`：不同节点返回的列表会合并，不是覆盖——这样每个节点只追加自己的部分
- 条件边：一个节点可以走向不同的下一个节点
- `compile()` 会把图编译成可执行的 Runnable

### 3. 对比：原生循环 vs LangGraph (1h)

| | 原生循环 | LangGraph |
|---|---|---|
| 状态管理 | 散落在 messages 列表 | State TypedDict，类型安全 |
| 流程控制 | `while True` + `break` | 图结构，可视化 |
| 条件路由 | `if-else` | 条件边，可测试 |
| 可恢复性 | 无 | Checkpoint 可暂停/恢复 |
| 面试价值 | "理解本质" | "用过工业级框架" |

**面试话术**："我先用原生 Python 写了一个 Agent 循环来理解 ReAct 原理，然后用 LangGraph 重构——因为审查流程是多步骤的（解析→定位→分析→汇总），StateGraph 比线性循环更适合管理状态转换。"
