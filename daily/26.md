### 1. 找测试数据 (1h)

挑 10 个不同复杂度、不同风格的 Python 文件：

```python
# scripts/benchmark.py
"""批量测试脚本"""
import asyncio
import json
import time
from pathlib import Path
from src.llm_client import LLMClient
from src.pipeline import ReviewPipeline
from src.schemas import ReviewRequest

# 测试用真实文件（从 GitHub 下载到 test_data/）
TEST_FILES = [
    "requests/api.py",        # HTTP 库 — 命名规范好
    "flask/app.py",           # Web 框架 — 代码量大
    "black/__init__.py",      # 格式化工具 — 风格典范
    "cookiecutter/main.py",   # CLI 工具 — 中等复杂度
    # ... 再加 6 个
]

async def bench():
    client = LLMClient(...)
    pipeline = ReviewPipeline(client)

    results = []
    for path in TEST_FILES:
        filepath = Path("test_data") / path
        if not filepath.exists():
            continue

        code = filepath.read_text()
        req = ReviewRequest(
            file_path=path,
            file_content=code,
            review_level="medium",
        )

        t0 = time.time()
        response, stats = await pipeline.review(req)
        elapsed = time.time() - t0

        results.append({
            "file": path,
            "lines": len(code.split('\n')),
            "issues_found": response.total_issues,
            "elapsed_s": round(elapsed, 2),
            "llm_calls": stats.llm_calls,
            "total_tokens": stats.total_tokens,
        })

    # 存结果
    with open("bench_results.json", "w") as f:
        json.dump(results, f, indent=2)

    # 汇总
    total_issues = sum(r["issues_found"] for r in results)
    total_time = sum(r["elapsed_s"] for r in results)
    total_calls = sum(r["llm_calls"] for r in results)
    print(f"Files: {len(results)}")
    print(f"Total issues: {total_issues}")
    print(f"Average time: {total_time/len(results):.1f}s")
    print(f"Total LLM calls: {total_calls}")
    print(f"Average issues/file: {total_issues/len(results):.1f}")

asyncio.run(bench())
```

### 2. 人工验证 (1.5h)

这是最关键的步骤——**面试官问"准确率多少"，你不能说"没测过"。**

对每个文件，逐条 issue 标记：
```
result = {
    "file": "requests/api.py",
    "issues": [...],
    "manual_review": {
        "total_reported": 12,
        "true_positives": 9,      # 确实有问题
        "false_positives": 2,     # 误报（Agent 太敏感）
        "false_negatives": 1,     # 漏报（你没检测到你已知的 bug）
        "notes": "误报：第45行的复杂度告警过于严格。漏报：第120行有未处理的异常。"
    }
}
```

### 3. 计算量化指标 (1h)

```python
# 汇总所有 10 个文件
accuracy = true_positives / (true_positives + false_positives)  # 准确率
recall = true_positives / (true_positives + false_negatives)    # 召回率
f1 = 2 * accuracy * recall / (accuracy + recall)

# 成本计算
avg_tokens_per_file = total_tokens / file_count
avg_cost_per_file = avg_tokens_per_file * price_per_token  # DeepSeek 价格
```

**目标指标**（达到任一就算成功）：

| 指标 | 合格 | 优秀 | 说明 |
|------|------|------|------|
| 准确率 | > 80% | > 90% | 不能瞎报 |
| 召回率 | > 60% | > 75% | 不能漏真 bug |
| 平均耗时 | < 15s | < 5s | 文件大小不同有波动 |
| 平均成本 | < ¥0.05/文件 | < ¥0.01/文件 | |
